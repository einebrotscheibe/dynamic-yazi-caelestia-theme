# [Yazi](https://github.com/sxyazi/yazi) Dynamic Theme System

A dynamic, automated theme engine for the **Yazi** terminal file manager. This system synchronizes your Yazi UI and syntax highlighting colors with your system-wide color scheme (generated by *caelestia*) in real-time using `systemd` path watchers.

---

## Features

- **Full UI Sync** — Updates borders, status bars, and file types dynamically.
- **Vibrant Syntax Highlighting** — Includes a `.tmTheme` template that ensures code previews match your terminal's energy.
- **Automated Updates** — Uses `systemd` to watch your color scheme file; as soon as you change your colors, Yazi updates instantly.
- **Flexible & Portable** — Support for custom Yazi configuration paths and varying color scheme locations.

---

## Prerequisites

- **[Yazi](https://github.com/sxyazi/yazi)** — The terminal file manager.
- **`jq`** — Used to parse your color scheme JSON.
- **`systemd`** — Required for the automated path-watching functionality.

---

## Quick Setup (Recommended)

The easiest way to get started is to use the provided `setup.sh` script. This script creates the necessary directories, copies the templates, configures the systemd units with your specific paths, and starts the watcher.

1. **Clone the repository:**
 ```bash
 git clone [Yazi](https://github.com/your-username/yazi-theme.git)
 cd yazi-theme
 ```

2. **Run the installer:**
 ```bash
 bash setup.sh
 ```

> **Note:** If you have a non-standard Yazi path or your color scheme is located elsewhere, edit the variables at the top of `setup.sh` before running it.

## Manual Installation

If you prefer to set things up yourself, follow these steps:

### 1. File Placement

Copy the templates and the updater script to your Yazi configuration folder:

```bash
mkdir -p ~/.config/yazi/theme
cp theme_template.toml ~/.config/yazi/
cp template_syntect.tmTheme ~/.config/yazi/theme/
cp updateColors.sh ~/.config/yazi/
chmod +x ~/.config/yazi/updateColors.sh
```

### 2. Configure Systemd

Copy the systemd units to your user service directory:

```bash
mkdir -p ~/.config/systemd/user
cp yazi-updater.path ~/.config/systemd/user/
cp yazi-updater.service ~/.config/systemd/user/
```

**Important:**
- Open `~/.config/systemd/user/yazi-updater.path` and ensure `PathModified` points to your `scheme.json`.
- Open `~/.config/systemd/user/yazi-updater.service` and ensure `ExecStart` points to your `updateColors.sh` script.

### 3. Enable the Watcher

Reload the systemd daemon and enable the path unit:

```bash
systemctl --user daemon-reload
systemctl --user enable --now yazi-updater.path
```

## How It Works

1. **The Templates** — The system uses `theme_template.toml` and `template_syntect.tmTheme`. These files contain placeholders like `{{PRIMARY}}` and `{{BG}}`.

2. **The Script** — `updateColors.sh` uses `jq` to pull hex codes from your `scheme.json`, then uses `sed` to swap the placeholders for real colors, generating `theme.toml` and `current_syntect.tmTheme`.

3. **The Watcher** — The `yazi-updater.path` unit monitors your `scheme.json`. Every time that file is saved, it triggers `yazi-updater.service`, which runs the script and refreshes your theme.

---

## Customization

You can tweak the look of the theme by editing `theme_template.toml`. The syntax highlighting logic can be adjusted in `theme/template_syntect.tmTheme`. If you change these templates, run the setup script again or wait for a color change to see the effects.
